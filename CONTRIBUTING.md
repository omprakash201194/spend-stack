# Contributing to SpendStack

Thank you for your interest in contributing to SpendStack.

This project aims to build a **privacy‑first personal finance
platform**, and contributions are welcome.

------------------------------------------------------------------------

## Development Workflow

1.  Fork the repository
2.  Create a feature branch
3.  Implement changes
4.  Write tests where appropriate
5.  Submit a Pull Request

Typical flow:

Feature Branch\
↓\
Pull Request\
↓\
CI Validation\
↓\
Code Review\
↓\
Merge

------------------------------------------------------------------------

## Coding Standards

-   Use **TypeScript**
-   Follow project ESLint rules
-   Use Prettier for formatting
-   Write clear commit messages

------------------------------------------------------------------------

## Issue Tracking

We organize work using:

-   **Epics**
-   **Stories**
-   **Tasks**
-   **Bugs**

Each issue contains:

-   Context
-   Goal
-   Scope
-   Acceptance Criteria

------------------------------------------------------------------------

## Pull Request Guidelines

A pull request should:

-   Reference the related issue
-   Include a summary of changes
-   Pass all CI checks
-   Include tests when applicable

Example PR template sections:

-   Summary
-   Related Issue
-   Changes
-   Testing

------------------------------------------------------------------------

## Areas Contributions Are Welcome

-   UI improvements
-   Parser improvements for banks
-   Documentation
-   Testing infrastructure
-   Performance improvements

------------------------------------------------------------------------

## Reporting Bugs

Please create a **bug issue** including:

-   Description
-   Steps to reproduce
-   Expected behavior
-   Environment details

------------------------------------------------------------------------

## Code of Conduct

Be respectful and constructive when collaborating.

We want SpendStack to be a welcoming open‑source project.
